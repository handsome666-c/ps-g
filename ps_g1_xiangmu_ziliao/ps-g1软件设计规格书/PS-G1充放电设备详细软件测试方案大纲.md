## 1. 测试概述

### 1.1 测试目标
- 验证软件功能符合需求规格书和软件规格书要求
- 确保软件在各种异常情况下的安全性和可靠性
- 覆盖所有代码路径和状态转换
- 验证保护机制的有效性
- 确保符合GB/T 18487.1-2023和GB/T 18487.4-2025标准

### 1.2 测试范围
- **代码层面**: 白盒测试、单元测试
- **模块层面**: 集成测试、接口测试
- **系统层面**: 功能测试、性能测试、安全测试
- **标准符合性**: 国标符合性测试

### 1.3 测试原则
- 测试驱动开发（TDD）
- 持续集成（CI）
- 测试覆盖率达标（语句覆盖率≥90%，分支覆盖率≥85%）
- 安全优先，故障导向测试
- 边界值和异常场景重点测试

## 2. 系统功能梳理

### 2.1 ASW（应用软件层）模块

#### 2.1.1 CP控制模块 (APP-001)
- **功能描述**: 控制导引功能，生成和检测CP信号
- **核心功能**:
  - PWM信号生成（1kHz，占空比可调）
  - CP电压检测（12V/9V/6V/异常电压）
  - CP状态机管理
  - CP异常处理
- **关键接口**:
  - `APP_CP_DetectCPState()` - 检测CP状态
  - `PwmIf_Set_Duty()` - 设置PWM占空比
  - `AdcM_get_CP_FB_AMP_update()` - 获取CP反馈电压

#### 2.1.2 充电功能模块 (APP-002)
- **功能描述**: 车辆充电功能管理
- **核心功能**:
  - 充电状态机（12个状态）
  - 充电流程控制
  - 充电参数管理
  - 充电异常处理
- **关键状态**:
  - 上电自检(POST) → 空闲(IDLE) → 插枪(CONNECTED) → 充电预检(PRECHECK) → 就绪(READY) → 充电中(CHARGING) → 结束(STOP)
  - 告警(WARNING)、故障(FAULT)、保护暂停(PPAUSE)、车端暂停(VPAUSE)

#### 2.1.3 放电功能模块 (APP-003)
- **功能描述**: 车辆放电功能管理（V2L/V2G）
- **核心功能**:
  - 放电状态机
  - 放电功率控制
  - 放电保护机制
- **关键状态**:
  - 上电自检 → 放电预检 → 放电中 → 保护暂停 → 故障

#### 2.1.4 故障诊断模块 (APP-004)
- **功能描述**: 故障检测、记录和管理
- **核心功能**:
  - 故障检测（过压、欠压、过流、过温、漏电、短路、继电器粘连等）
  - DTC管理
  - 冻结帧记录
  - 故障清除

#### 2.1.5 灯语显示模块 (APP-005)
- **功能描述**: LED状态指示
- **核心功能**:
  - LED颜色控制（蓝、绿、红、黄）
  - 闪烁模式控制（常亮、闪烁、呼吸）
  - 状态映射

#### 2.1.6 电压测量模块 (APP-006)
- **功能描述**: 输入/输出电压检测
- **核心功能**:
  - 电压采样
  - 电压滤波
  - 过压/欠压检测
  - 校准支持

#### 2.1.7 电流测量模块 (APP-007)
- **功能描述**: 充电/放电电流检测
- **核心功能**:
  - 电流采样
  - 电流滤波
  - 过流检测（三段保护）
  - 校准支持

#### 2.1.8 校准功能模块 (APP-008)
- **功能描述**: 电压电流校准
- **核心功能**:
  - 电压校准
  - 电流校准
  - 校准数据存储
  - 恢复出厂设置

#### 2.1.9 日志功能模块 (APP-009)
- **功能描述**: 运行日志记录
- **核心功能**:
  - 运行日志记录
  - 故障日志存储
  - 日志查询
  - 日志导出

#### 2.1.10 工厂模式模块 (APP-010)
- **功能描述**: 生产测试和参数配置
- **核心功能**:
  - 工厂模式进入/退出
  - 生产测试
  - 参数配置
  - 出厂测试

### 2.2 BSW（基础软件层）模块

- Wdg - 看门狗驱动

- Uart - UART驱动
- Port - 端口驱动
- Dio - DIO驱动
- PWM - PWM驱动
- Adc - ADC驱动

### 2.4 关键保护机制

#### 2.4.1 充电保护
- **过压保护**: 电压>275V触发，265V恢复
- **欠压保护**: 电压<80V触发，90V恢复
- **过流一段**: 超过设定值10-12A持续5-10秒
- **过流二段**: 超过设定值12-18A持续3秒
- **过流三段**: ≥18A，100ms内切断
- **MCU过温**: >100℃持续5秒
- **NTC1过温**: >100℃持续5秒
- **漏电保护**: 连续5ms高电平
- **短路保护**: ADC采样值<2865(3.5V)

#### 2.4.2 放电保护
- **过压保护**: 电压>264V触发，254V恢复
- **欠压保护**: 电压<176V触发，186V恢复
- **过流一段**: 超过35A持续5-10秒
- **过流二段**: 超过40A，100ms内切断
- **温度保护**: 同充电保护

## 3. 测试策略与方法论

### 3.1 测试金字塔
```
         /\
        /  \
       /系统\      - 黑盒测试、端到端测试
      /______\

   /   单元    \  - 白盒测试、函数级测试
  /____________\

### 3.2 白盒测试策略
- **目标**: 验证代码内部逻辑的正确性
- **方法**:
  - 语句覆盖: 每条语句至少执行一次
  - 分支覆盖: 每个判断分支至少执行一次
  - 路径覆盖: 覆盖所有可能的执行路径
  - 条件覆盖: 每个条件的真假都测试
- **工具**: 静态代码分析、动态代码覆盖率工具

### 3.3 单元测试策略
- **目标**: 验证单个函数/模块的功能正确性
- **方法**:
  - 独立测试每个函数
  - 使用Mock/Stub隔离依赖
  - 测试正常流程和异常流程
  - 边界值测试
- **框架**: Unity (C) / Google Test (C++)

### 3.4 集成测试策略
- **目标**: 验证模块间接口和交互
- **方法**:
  - 自底向上集成
  - 接口测试
  - 数据流测试
  - 通信测试
- **重点**: 模块间的数据传递、状态同步

### 3.5 系统测试策略
- **目标**: 验证完整系统的功能
- **方法**:
  - 黑盒测试
  - 端到端测试
  - 场景测试
  - 用户验收测试

## 4. 白盒测试方案

### 4.1 白盒测试目标
- 代码语句覆盖率 ≥ 90%
- 分支覆盖率 ≥ 85%
- 路径覆盖率 ≥ 70%
- 条件覆盖率 ≥ 80%

### 4.2 静态代码分析

#### 4.2.1 编码规范检查
- **工具**: PC-lint / Cppcheck
- **检查项**:
  - MISRA-C:2012规范符合性
  - 代码风格一致性
  - 未使用变量/函数
  - 潜在的内存泄漏
  - 数组越界风险
  - 未初始化变量

#### 4.2.2 复杂度分析
- **圈复杂度**: 每个函数≤15
- **认知复杂度**: 每个函数≤20
- **嵌套深度**: 不超过4层

#### 4.2.3 代码度量
- **函数长度**: 不超过50行
- **文件长度**: 不超过1000行
- **注释率**: ≥20%

### 4.3 动态白盒测试

#### 4.3.1 代码覆盖率测试
- **语句覆盖策略**:
  - 识别所有可执行语句
  - 设计测试用例覆盖每条语句
  - 重点关注异常处理路径
  
- **分支覆盖策略**:
  - 识别所有if/else、switch/case
  - 设计测试用例覆盖每个分支
  - 重点关注边界条件判断
  
- **路径覆盖策略**:
  - 绘制控制流图
  - 识别所有独立路径
  - 设计测试用例覆盖关键路径

#### 4.3.2 关键函数白盒测试

**CP控制模块关键函数**:
```
函数1: CP_GeneratePWM()
- 输入: 占空比值
- 分支: 
  - 占空比<6% → 使用默认值
  - 6%≤占空比≤100% → 正常生成
  - 占空比>100% → 使用100%
- 测试点: 
  - 占空比=0%, 5%, 6%, 50%, 100%, 105%
  - PWM频率验证
  - 上升/下降沿时间

函数2: CP_DetectAmplitude()
- 输入: CP电压值
- 分支:
  - 电压≥12V且≤13.2V → STATE_A (12V状态)
  - 电压≥9V且≤10.8V → STATE_B (9V状态)
  - 电压≥6V且≤7.2V → STATE_C (6V状态)
  - 其他 → INVALID
- 测试点:
  - 电压=10.5V, 12V, 12.5V, 9V, 9.5V, 6V, 7V
  - 边界值: 7.8V, 8.2V, 10.2V, 10.8V, 13.2V, 13.5V
```

**充电状态机白盒测试**:
```
函数: Charging_StateMachine()
- 状态转换覆盖:
  - POST → IDLE (自检成功)
  - POST → FAULT (自检失败)
  - IDLE → CONNECTED (插枪)
  - IDLE → WARNING (CP异常)
  - CONNECTED → IDLE (拔枪)
  - CONNECTED → PRECHECK (无鉴权启充)

  - CHARGING → PPAUSE (保护触发)
  - CHARGING → IDLE (拔枪)
  - VPAUSE → CHARGING (车机恢复)
  - VPAUSE → IDLE (拔枪)
  - PPAUSE → READY (恢复)
  - PPAUSE → IDLE (拔枪)
  
- 测试方法:
  - 状态转换矩阵测试
  - 无效状态转换测试
  - 状态转换条件测试
```

**保护函数白盒测试**:
```

- 输入: 电压值

## 12. 缺陷管理

### 12.1 缺陷分级
```
P0 - 致命: 系统崩溃、安全失效、无法使用
  - 响应时间: 2小时
  - 修复时间: 24小时

P1 - 严重: 主要功能失效、数据丢失
  - 响应时间: 4小时
  - 修复时间: 3天

P2 - 一般: 次要功能失效、UI问题
  - 响应时间: 8小时
  - 修复时间: 1周

P3 - 轻微: 文字错误、建议性改进
  - 响应时间: 24小时
  - 修复时间: 2周
```

### 12.2 回归测试策略
```
- P0/P1缺陷: 完整回归测试
- P2缺陷: 相关功能回归
- P3缺陷: 累积处理
- 每次发布前: Smoke Test
```
